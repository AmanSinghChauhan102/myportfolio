Selection Sort Algorithm

What is Selection Sort?

Selection Sort is a simple comparison-based sorting algorithm that divides the input list into two parts: a sorted sublist and an unsorted sublist. The algorithm repeatedly selects the smallest (or largest) element from the unsorted sublist and moves it to the end of the sorted sublist.

Algorithm Steps

  1. Set the first element as the minimum (or maximum).
  2. Compare the minimum with the next element in the array.
  3. If any element is smaller than the current minimum, set that element as the new minimum.
  4. Swap the minimum with the first element of the unsorted part.
  5. Repeat the process for the remaining unsorted portion until the entire array is sorted.
C 
#include<stdio.h>

int main()
{
    int list[] = {12, 7, 19, 6, 5}; 
    int size = 5;
    
    printf("Original List : ");
    for(int i = 0; i < size; i++)
    {
        printf("%d ", list[i]);
    }
    printf("\n");
    

    for( int i = 0; i < size; i++ )
    {
        int select = i;
        for (int j = i + 1; j < size; j++)
        {
            if( list[j] < list[select] )
            {
                select = j;
            }
        }

        int temp = list[i];
        list[i] = list[select];
        list[select] = temp;
    }

    printf("Sorted List : ");
    for(int i = 0; i < size; i++)
    {
        printf("%d ", list[i]);
    }
    printf("\n");
    
    return 0;
}

Code Explanation

The provided C program demonstrates the Selection Sort algorithm:

  • Initialization: An array list[] is initialized with unsorted values.
  • Outer Loop: The outer loop runs from the first element to the last element of the array.
  • Selection: The inner loop finds the minimum element in the remaining unsorted array.
  • Swapping: After finding the minimum element, it is swapped with the first element of the unsorted part.
  • Result: After all iterations, the array is sorted in ascending order.

Time Complexity

Case Time Complexity
Best Case O(n²)
Average Case O(n²)
Worst Case O(n²)

Why O(n²)?

Selection Sort has a time complexity of O(n²) in all cases because:

  • It performs n comparisons in the first pass, n-1 in the second pass, and so on.
  • The total number of comparisons is n*(n-1)/2, which is O(n²).
  • Even if the array is already sorted, the algorithm still performs all comparisons.
